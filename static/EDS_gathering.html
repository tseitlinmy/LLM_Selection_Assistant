LLM call Evaluation Dataset data gathering

Problem statement

We have LLM call which uses specified model and it works right. We'll name this model as "initial model" and this LLM call as "LLM call". But we are not satisfied initial model's characteristics: "response time" or cost.

What is Evaluation Dataset

We want to select another model that offers the best balance of performance and cost for a given LLM call. To do this, we should rely on the actual prompts that this call will send to the LLM. This set of prompts (we will call it the "Evaluation Dataset") should be sufficiently representative to yield trustworthy statistics for each tested model. We will calculate key metrics: "average response time" and "average cost per prompt" and compare them to select the model with the characteristics best suited to our needs.

Tools support

Concerning tools usage - there may be following possibilities:

  • given LLM call does not use tools at all
  • the LLM call uses tools explicitly
  • the initial model calls built-in tools implicitly

Correspondingly:

  • To support explicit tools calling in evaluation, need to have tools calling dump for each prompt in Evaluation Dataset.
  • To support implicit built-in tools calling in evaluation, need to detect which built-in tools are used and to use exclusively those models that support these tools.

Then answer for each prompt from the Evaluation Dataset should contain information about built-in tools calling and dump of explicit tools calling (if this LLM call has tools list).

Evaluation Dataset preparation

  1. Create dataset directory (mkdir dataset). All Evaluation Dataset files will be saved in this directory.
  2. Save system prompt of this LLM call into system_prompt.txt
  3. Now we need to fix system prompt of this LLM call to dump usage of tools.
    • If this LLM call uses external tools: prefix the system prompt with "Tools dumping" prompt prefix. Expand "Tools dumping" prompt prefix item in the application and press "Copy to clipboard" button – the prefix in clipboard now. You may use it.
    • If this LLM call does not use external tools – it may use built-in tools of the initial model anyway. So prefix the system prompt with "LLM Built-In Tools checking" prompt prefix. Expand "LLM Built-In Tools checking" prompt prefix item in the application and press "Copy to clipboard" button – the prefix in clipboard now. You may use it.
  4. After the system prompt was fixed, launch set of user prompts (~ 20-50 prompts) on this LLM call.
    • You need save each user prompt in files: user_prompt00.txt, user_prompt01.txt, user_prompt02.txt, ...
    • You need save each LLM answer in files: answer00.txt, answer01.txt, answer02.txt, ...
  5. Compress dataset directory into zip archive file (dataset.zip)
  6. Upload dataset.zip to application by 'Upload "Evaluation Set" File' button pressing.

Press "Evaluate" button in the application to start dataset evaluation process.
